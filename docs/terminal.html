阶段 01

快速开始

打开你的第一个终端，认识它的基本构成。

1.1

如何打开终端

macOS 上有三种方式打开终端

🔍 Spotlight 搜索

按下 ⌘ + 空格，输入 "Terminal" 或 "终端"，回车打开。这是最快的方式。

📁 访达 (Finder)

打开 Finder → 应用程序 → 实用工具 → 终端。适合不熟悉快捷键的新手。

💻 iTerm2 推荐

更强大的终端替代品。访问 iterm2.com 下载，功能远超系统终端。

💡 推荐：安装 iTerm2 作为默认终端。它支持分屏、丰富的主题、更好的颜色渲染，是专业用户的首选。
1.2

认识终端界面

了解提示符的组成部分

zsh — 终端界面解析
# 典型的 zsh 提示符由以下部分组成：
用户名 @ 主机名 : ~/当前目录 ❯ 命令在这里输入
viper@MacBook-Pro:~/projects ❯
# ❯ 符号表示终端在等待你输入命令
# ~ 代表你的主目录 (/Users/你的用户名)
# 按 Tab 键可以自动补全命令和路径
# 按 ↑↓ 箭头键可以浏览历史命令

阶段 02

核心命令大全

掌握这些命令，覆盖 90% 的日常终端操作。

2.1

文件与目录操作

mkdir — 创建目录
mkdir
Make Directory — 创建新文件夹
❯ mkdir my-project
❯ mkdir -p a/b/c
# -p 递归创建多层目录
touch — 创建文件
touch
创建空文件，或更新文件的修改时间
❯ touch index.html
❯ touch a.js b.js c.js
# 一次创建多个文件
cp — 复制
cp
Copy — 复制文件或目录
❯ cp file.txt file-backup.txt
❯ cp -r folder/ folder-backup/
# -r 递归复制整个目录
mv — 移动/重命名
mv
Move — 移动文件，也用于重命名
❯ mv old-name.txt new-name.txt
❯ mv file.txt ~/Documents/
rm — 删除
rm
Remove — 删除文件或目录（操作不可撤销！）
❯ rm file.txt
❯ rm -rf folder/
# -r 递归 -f 强制，不提示确认
find — 搜索文件
find
在目录树中查找文件
❯ find . -name "*.js"
❯ find ~ -mtime -7
# 最近 7 天修改的文件
⚠️ 警告：rm -rf 删除的文件不会进入废纸篓，无法恢复。执行前请三思，永远不要运行 rm -rf / 或 rm -rf ~！

阶段 03

macOS 专属配置

打造专业的 macOS 开发环境，让终端更顺手更强大。

🍺 Homebrew 包管理器

macOS 上最重要的工具。用一条命令安装、更新、卸载几乎所有开发工具和应用。

安装与使用 Homebrew
# 1. 安装 Homebrew（复制到终端执行）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
# 2. 常用 brew 命令
❯brew install git
❯brew install node
❯brew install gh # GitHub CLI
❯brew upgrade # 更新所有软件
❯brew list # 已安装列表

⚙️ zsh 配置 macOS 默认

macOS 从 Catalina 起默认使用 zsh。配置 ~/.zshrc 文件来自定义你的终端环境。

~/.zshrc 配置示例
# 打开配置文件编辑
❯nano ~/.zshrc

# PATH 配置
export PATH="/opt/homebrew/bin:$PATH"
# 常用别名
alias ll='ls -la'
alias gs='git status'
alias dev='npm run dev'

❯source ~/.zshrc # 重载配置

🎨 oh-my-zsh 框架

强大的 zsh 配置管理框架，提供数百个主题和插件，让终端焕然一新，功能强大。

安装 oh-my-zsh
sh -c "$(curl -fsSL https://ohmyz.sh/install.sh)"
# 推荐插件（.zshrc 中配置）
plugins=(git
   zsh-autosuggestions
   zsh-syntax-highlighting
   z web-search)
ZSH_THEME="powerlevel10k/powerlevel10k"

🍎 macOS 独有命令 专属

这些命令只在 macOS 上可用，非常实用，是与 Finder 和系统交互的利器。

macOS 专属命令
# 用默认程序打开文件/目录/URL
❯open README.md
❯open . # 在Finder中打开当前目录
❯open https://github.com

# 剪贴板操作
❯cat file.txt | pbcopy

🚀 推荐工具链： iTerm2 + oh-my-zsh + powerlevel10k + zsh-autosuggestions + zsh-syntax-highlighting — 这是 macOS 开发者最流行的终端配置组合，安装后终端体验质的飞跃。
阶段 04

进阶技巧

掌握这些特性，让你的终端效率再上一个台阶。

管道 |

将一个命令的输出传递给另一个命令，这是终端最强大的特性之一。

❯ ls -la | grep ".js"
# 列出所有 .js 文件

❯ cat log.txt | grep "error" | wc -l
# 统计错误行数

❯ ps aux | grep node | awk '{print $2}'
# 获取 node 进程 PID

重定向 > >>

将命令输出保存到文件，或从文件读取输入。

❯ ls > files.txt
# 覆盖写入文件

❯ echo "new line" >> log.txt
# 追加写入

❯ command 2> error.log
# 错误输出重定向

❯ command &> all.log
# 所有输出到文件

alias 别名

为常用的长命令创建短别名，大幅提升输入效率。

# 添加到 ~/.zshrc

alias ll='ls -la --color'
alias gs='git status'
alias gp='git push'
alias ni='npm install'
alias dev='npm run dev'

# 查看所有别名
❯ alias

环境变量

配置系统环境，管理 API 密钥和工具路径。

# 查看环境变量
❯ printenv
❯ echo $HOME
❯ echo $PATH

# 设置临时变量（当前会话）
export API_KEY="your_key_here"

# 永久设置（写入 ~/.zshrc）
export ANTHROPIC_KEY="sk-..."

Shell 脚本

将多条命令写入脚本文件，实现自动化任务执行。

#!/bin/zsh # 脚本文件头
# deploy.sh

echo "🚀 开始部署..."
git pull origin main
npm install
npm run build
echo "✅ 部署完成！"

# 赋予执行权限并运行
❯ chmod +x deploy.sh
❯ ./deploy.sh

SSH 与远程

使用 SSH 连接远程服务器，配置密钥免密登录。

# 生成 SSH 密钥对
❯ ssh-keygen -t ed25519

# 复制公钥到剪贴板
❯ cat ~/.ssh/id_ed25519.pub | pbcopy
# 然后粘贴到 GitHub Settings

# 连接服务器
❯ ssh user@server.com

# 测试 GitHub SSH 连接
❯ ssh -T git@github.com

GitHub Copilot CLI

在终端中直接向 GitHub Copilot 提问。用自然语言描述想要的命令，AI 帮你生成并解释，无需记忆复杂语法。

gh extension
gh copilot 使用示例
# 安装 GitHub CLI 和 Copilot 扩展
❯brew install gh
❯gh auth login
❯gh extension install github/gh-copilot

# 用自然语言获取命令建议
❯gh copilot suggest "找出占用8080端口的进程"
lsof -i :8080

# 解释命令的含义
❯gh copilot explain "find . -name '*.log' -mtime +7"
查找当前目录下所有超过 7 天未修改的 .log 文件
✦

Claude Code

Anthropic 推出的终端 AI 编程助手。直接在项目目录中运行，理解整个代码库上下文，支持读写文件和执行命令。

Anthropic
Claude Code 使用
# 安装 Claude Code
❯npm install -g @anthropic-ai/claude-code

# 在项目目录启动
❯cd ~/my-project
❯claude
✓ Claude Code 已启动

你: 帮我给这个 API 添加错误处理

GitHub CLI (gh)

在终端中管理 GitHub 的一切：PR、Issues、Actions、Releases。与 Copilot 深度集成，是现代开发工作流的核心。

AI 辅助工作流

利用 AI 帮你写复杂的 Shell 脚本、解决报错、优化命令。掌握正确的提问方式，让 AI 成为你的终端搭档。

最佳实践
AI 辅助终端技巧
# 遇到报错，让 AI 解释
❯npm install 2>&1 | pbcopy
然后粘贴错误到 Claude/Copilot 询问解决方案

# 用 AI 生成复杂命令
"批量压缩当前目录所有 PNG 图片"

# 提问技巧：提供上下文
✓ "在 macOS 上，如何用 zsh 脚本..."

🔑 安全原则：永远不要盲目执行 AI 生成的命令，特别是涉及 sudo、rm 等操作。先用 gh copilot explain 或询问 Claude 理解命令含义，再决定是否执行。
